Payments

The Client instance provides a payments object to interact directly with Quickpay payments.

The available functions are listed below with descriptions, examples, arguments and expected return values.

tip

Make sure that you have a correct setup with a client based on the usage page

get​

Fetch all data for a specific payment

  • Example
  • Arguments
  • Return
$response = $client->payments->get($id);
nametyperequireddefault
idnumbertrue-

Instance of Psr\Http\Message\ResponseInterface with raw data from Quickpay API.

create​

Creates a new payment. This is rarely used alone, since it will only create a payment in Quickpay, but without any way for the user to submit payments.

Please refer to the createLink function below, which will both create the payment and return a link for the user to use for payments.

  • Example
  • Arguments
  • Return
$response = $client->payments->create($input);
nametyperequireddefault
inputPaymentInputtrue-

Instance of Psr\Http\Message\ResponseInterface with raw data from Quickpay API.

createLink​

Creates a new payment and a link to the payment form to be used for paying.

  • Example
  • Arguments
  • Return
$response = $client->payments->createLink($input);
nametyperequireddefault
inputPaymentLinkInputtrue-

Instance of Psr\Http\Message\ResponseInterface with raw data from Quickpay API.

updateLink​

Updates an existing payment link.

  • Example
  • Arguments
  • Return
$response = $client->payments->updateLink($id, $input);
nametyperequireddefault
idnumbertrue-
inputPaymentLinkInputtrue-

Instance of Psr\Http\Message\ResponseInterface with raw data from Quickpay API.

capture​

Captures a partial or full amount of an existing payment.

  • Example
  • Arguments
  • Return
$response = $client->payments->capture($id, $amount);
nametyperequireddefaultdescription
idnumbertrue-
amountnumberfalsenullIf not provided, it will attempt to capture the full amount of the payment

On success

Instance of Psr\Http\Message\ResponseInterface with raw data from Quickpay API.

On failure

null
